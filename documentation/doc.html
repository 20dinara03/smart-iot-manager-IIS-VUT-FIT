Implementace

Struktura projektu

db: Kód pro vytvoření databáze.
dbenv: Konfigurační soubory pro databázi a proměnné prostředí.
nats: Nastavení pro server NATS pro výměnu zpráv.
scripts: Kód pro práci s metrikami, včetně generování, zpracování a zobrazování.
src/grafita: Backend logika, datové modely a API. /templates/grafs obsahuje frontendové pohledy pro grafy.
src/templates: Všechny soubory pohledů pro frontend.
src/core: Klíčové soubory pro konfiguraci a nastavení projektu v Django.
src/mq: Kód pro Django aplikaci "mq".
src/rest: Django aplikace pro práci s REST API.

Hlavní soubory pro implementaci funkčnosti projektu jsou umístěny ve složce src:

Soubor apps.py

Soubor apps.py obsahuje konfiguraci aplikace Django. Toto definuje třídu GrafitaConfig, která specifikuje použití BigAutoField jako výchozí pole pro pole auto ID ve vaší aplikaci ?grafita?.

Soubor kpi.py

Soubor kpi.py poskytuje základní třídy pro implementaci KPI (Key Performance Indicators) a jejich registraci. Třída AbstractKPI je abstraktní a obsahuje dvě abstraktní metody: validate a val. Třídy Vyšší a Dolní jsou implementace klíčových ukazatelů výkonu, které kontrolují, zda je hodnota metriky nad nebo pod určitou prahovou hodnotou. Soubor také obsahuje komentovaný kód pro Delta KPI, který pravděpodobně potřebuje nějakou práci.

Soubor models.py

Soubor models.py obsahuje definice modelu Django pro vaši aplikaci. Tento soubor popisuje datové modely, jako jsou DevicesGroup, DeviceType, DeviceTypeParameter, Device, KPI a Metric. Tyto modely představují různé typy dat a vzájemně se ovlivňují prostřednictvím cizích klíčů a vztahů many-to-many.

Soubor urls.py

Soubor urls.py definuje trasy URL pro vaši aplikaci. Soubor je strukturován podle dílčích aplikací (např. zařízení, typy zařízení, skupiny zařízení atd.), z nichž každá má své vlastní adresy URL a přidružené třídy zobrazení Django. Zahrnuty jsou také adresy URL pro různé akce, jako je mazání, aktualizace a vytváření zařízení a typů zařízení, a také adresy URL pro autorizaci.

Příslušné soubory ve views

V rámci složky views existují několik souborů pro implementaci specifické funkcionality aplikace. Každý soubor obsahuje odpovídající pohledy pro danou část funkcí:

  • auth.py: Obsahuje pohledy pro ověření uživatele (přihlášení a registrace). Implementuje formuláře pro přihlášení a registraci.
  • device.py: Spravuje operace CRUD pro zařízení. Používá formuláře Django pro operace související se zařízením. Zahrnuje zobrazení pro vytváření, aktualizaci a mazání zařízení. Implementuje funkci sdílení pro zařízení.
  • device_groups.py: Spravuje operace CRUD pro skupiny zařízení. Implementuje formuláře pro vytváření a aktualizaci skupin zařízení. Poskytuje pohledy pro zobrazení skupin zařízení, jejich sdílení a zpracování požadavků na přístup ke skupinám.
  • device_types.py: Spravuje operace CRUD pro typy zařízení. Implementuje formuláře pro vytváření a aktualizaci typů zařízení a jejich parametrů.
  • groups.py: Zpracovává skupiny uživatelů a oprávnění. Poskytuje pohledy pro správu skupin uživatelů, přidávání/odebírání oprávnění a odstraňování skupin.
  • mixins.py: Obsahuje mixiny pro zpracování autentizace uživatelů a přístupu personálu.
  • users.py: Spravuje pohledy související s uživateli. Implementuje zobrazení pro výpis uživatelů a zobrazení podrobností o uživateli. Umožňuje aktualizaci uživatelských informací a správu uživatelských skupin.
